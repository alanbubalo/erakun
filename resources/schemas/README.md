# UBL / EN 16931 / HR-CIUS Validation Schemas

Vendored validation artifacts used by `App\Validation\UblValidator`. Everything in this tree is a copy of an upstream artifact, downloaded once and committed; runtime never fetches anything.

## Layout

```
resources/schemas/
├── ubl/              # OASIS UBL 2.1 XSDs (Croatian-curated bundle)
├── en16931/          # CEN EN 16931 Schematron (UBL syntax binding)
├── hr-cius/          # HR-CIUS-EXT-EN16931 Schematron (Croatian extensions)
├── skeleton/         # ISO Schematron skeleton (compiles .sch → .xsl)
├── saxon/            # Saxon-HE 12.5 + xmlresolver (XSLT 2.0 engine)
└── compiled/         # Generated by `php artisan schema:compile` (committed)
```

## Sources & Versions

| Folder | Source | Version |
|---|---|---|
| `ubl/` | [Porezna uprava — UBL 2.1 eRačun](https://porezna.gov.hr/fiskalizacija/api/dokumenti/198) | 16-Mar-2026 |
| `en16931/` | [ConnectingEurope/eInvoicing-EN16931](https://github.com/ConnectingEurope/eInvoicing-EN16931) — `ubl/schematron/` | `master` |
| `hr-cius/` | [Porezna uprava — HR-UBL Schematron](https://porezna.gov.hr/fiskalizacija/api/dokumenti/197) | 13-Mar-2026 |
| `skeleton/` | [Schematron/schematron](https://github.com/Schematron/schematron) — `trunk/schematron/code/` | `master` |
| `saxon/` | [Saxon-HE on Maven Central](https://repo1.maven.org/maven2/net/sf/saxon/Saxon-HE/) | 12.5 |

The `ubl/` bundle is the **Porezna-curated** UBL 2.1 set (their reference validator's input). It includes `HRExtensionAggregateComponents-1.xsd` alongside the standard OASIS XSDs. Using this set keeps us byte-aligned with the official Croatian validator.

## Runtime Dependency

Validation runs Saxon-HE via `java`. Required: **Java 11+** on `PATH`. The Saxon jars under `saxon/` are the only Saxon installation needed — no system-wide install. The directory is **gitignored**; `php artisan schema:install` fetches the jars from Maven Central with SHA-1 verification, and `composer setup` runs it automatically after a fresh clone.

## Regenerating `compiled/`

```
php artisan schema:compile
```

This runs each `.sch` through the ISO Schematron skeleton's three-stage pipeline (`iso_dsdl_include` → `iso_abstract_expand` → `iso_svrl_for_xslt2`) and writes the resulting XSLT 2.0 validators to `compiled/`.

`compiled/` is **gitignored** — the files are deterministic build artifacts, fully reproducible from the vendored `.sch` + skeleton. `composer setup` runs `schema:compile` automatically; otherwise run it manually after cloning, and re-run any time a `.sch` changes (re-vendoring an updated EN 16931 release or HR-CIUS revision).

## Phase Filtering

`compiled/en16931.xsl` is built with `phase=EN16931model_phase`. EN 16931's UBL **syntax** pattern (`UBL-CR-*`) bans constructs that HR-CIUS extensions legitimately require — `cac:SellerContact`, `cbc:IssueTime`, `ext:UBLExtensions` for signatures. Compiling the model phase only means BR-* business rules (BR-CO-10 et al.) fire while syntax restrictions are skipped. EN 16931 codelists are also skipped because HR-CIUS bundles its own (`HR-CIUS-EXT-EN16931-UBL-codes.sch`).

## Re-vendoring Steps

**UBL XSDs and HR-CIUS Schematron** (Porezna documents 197 and 198 — the IDs are stable; check the [eRačun page](https://porezna.gov.hr/fiskalizacija/bezgotovinski-racuni/eracun) for the current document IDs if URLs change):

```
curl -sL "https://porezna.gov.hr/fiskalizacija/api/dokumenti/198" -o ubl-bundle.zip
curl -sL "https://porezna.gov.hr/fiskalizacija/api/dokumenti/197" -o hr-cius-bundle.zip
```

Extract and copy XSDs/`.sch` files into the matching subdirectories.

**EN 16931 Schematron**:

```
BASE="https://raw.githubusercontent.com/ConnectingEurope/eInvoicing-EN16931/master/ubl/schematron"
curl -sL "$BASE/EN16931-UBL-validation.sch"        -o en16931/EN16931-UBL-validation.sch
curl -sL "$BASE/abstract/EN16931-model.sch"        -o en16931/abstract/EN16931-model.sch
curl -sL "$BASE/abstract/EN16931-syntax.sch"       -o en16931/abstract/EN16931-syntax.sch
curl -sL "$BASE/UBL/EN16931-UBL-model.sch"         -o en16931/UBL/EN16931-UBL-model.sch
curl -sL "$BASE/UBL/EN16931-UBL-syntax.sch"        -o en16931/UBL/EN16931-UBL-syntax.sch
curl -sL "$BASE/codelist/EN16931-UBL-codes.sch"    -o en16931/codelist/EN16931-UBL-codes.sch
```

**ISO Schematron skeleton**:

```
BASE="https://raw.githubusercontent.com/Schematron/schematron/master/trunk/schematron/code"
curl -sL "$BASE/iso_dsdl_include.xsl"               -o skeleton/iso_dsdl_include.xsl
curl -sL "$BASE/iso_abstract_expand.xsl"            -o skeleton/iso_abstract_expand.xsl
curl -sL "$BASE/iso_svrl_for_xslt2.xsl"             -o skeleton/iso_svrl_for_xslt2.xsl
curl -sL "$BASE/iso_schematron_skeleton_for_saxon.xsl" -o skeleton/iso_schematron_skeleton_for_saxon.xsl
```

**Saxon-HE** (one-time, with SHA-1 verification):

```
php artisan schema:install
```

After re-vendoring any `.sch` or skeleton file, run `php artisan schema:compile` and commit the resulting `compiled/*.xsl`.
